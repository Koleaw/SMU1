# Pages CMS: подготовка конфигурации

## Что такое `.pages.yml` в проекте
`.pages.yml` — это конфигурационный файл Pages CMS в корне репозитория. По нему Pages CMS понимает, какие коллекции/файлы можно редактировать, какие поля показывать в форме и куда складывать загружаемые файлы.

## Какие коллекции доступны для редактирования
В текущей конфигурации доступны:

### Каталог
- Разделы продукции (`src/content/product-sections`)
- Категории изделий (`src/content/product-categories`)
- Изделия (`src/content/products`)

### Компания
- Услуги (`src/content/services`)
- Выполненные объекты (`src/content/projects`)
- Вакансии (`src/content/jobs`)

### Настройки
- Настройки сайта (`src/content/site-settings/global.json`)

## Куда загружаются изображения
Загрузка файлов настроена локально в репозиторий:
- физическое сохранение: `public/uploads`
- путь в JSON-контенте: `/uploads/<имя_файла>`

## Как добавить новый тип изделий (пример: «Фонари»)
1. Открыть в Pages CMS коллекцию **«Категории изделий»**.
2. Создать новую запись с полями категории (включая `slug`, `parentSectionSlug`, `mode`, `isActive`, `showInSectionGrid`).
3. Сохранить изменения — будет создан/обновлен JSON-файл в `src/content/product-categories`.
4. При необходимости затем добавить конкретные позиции в коллекции **«Изделия»**, привязав их через `productCategorySlug`.

> В этом этапе новые сущности в проект не добавляются — это инструкция для будущего контент-редактирования.

## Как добавить новую позицию изделия
1. Открыть коллекцию **«Изделия»**.
2. Создать запись и заполнить обязательные поля: `title`, `slug`, `productCategorySlug`, `shortDescription`, `priceMode`, `order`, `isActive`, `showInCatalog`.
3. Если нужно, добавить цену (`priceFrom`), изображение (`image`), SEO-поля.
4. Сохранить запись.

## Как добавить вакансию
1. Открыть коллекцию **«Вакансии»**.
2. Создать запись и заполнить обязательные поля: `title`, `slug`, `city`, `employmentType`, `shortDescription`, `responsibilities`, `requirements`, `conditions`, `order`, `isActive`.
3. Сохранить запись.

## Как скрыть вакансию
- Для скрытия вакансии установить `isActive: false` в записи вакансии и сохранить.

## Как скрыть изделие
- Вариант 1: установить `isActive: false`.
- Вариант 2: оставить `isActive: true`, но установить `showInCatalog: false`.

## Что останется сделать для реального входа в админку
Текущий этап включает только схему (`.pages.yml`). Для полноценного рабочего входа в админку позже отдельно потребуется:
- выбрать и настроить способ аутентификации в Pages CMS;
- выдать доступы редакторам к репозиторию/ветке;
- согласовать процесс публикации (кто и как мержит изменения).

В этом этапе намеренно **не** добавлялись: `/admin`, GitHub App, self-host, secrets/env, деплойные доработки.
