# PAGE BLOCKS Stage 1 (пилот)

## Что такое `pageBlocks`
`pageBlocks` — это массив заранее типизированных блоков в content layer (`src/content/product-sections/*.json`), из которого рендерится страница проектного направления.

В Stage 1 поле добавлено как **optional**, чтобы страницы без `pageBlocks` продолжали работать по старой логике без миграции.

## Область пилота
В этом этапе пилот включен только для страницы:
- `/navesy/` (реальный slug: `navesy-i-kozyrki`)

Остальные страницы не мигрируются на `pageBlocks` в рамках Stage 1.

## Доступные типы блоков
Разрешены только следующие `type`:
- `cardGrid`
- `listPanel`
- `mediaText`
- `process`
- `factorList`
- `notice`
- `exampleGrid`

Каждый блок управляется полями `order` и `isActive`.

## Как менять порядок блоков
1. Откройте запись страницы в Pages CMS (`product-sections` → `navesy`).
2. В массиве `pageBlocks` измените поле `order` у нужных блоков.
3. Сохраните запись — рендерер сортирует блоки по `order` по возрастанию.

## Как скрыть блок
Установите `isActive: false` у блока.

Рендерер пропускает:
- блоки, у которых `isActive === false`;
- блоки без контента;
- неизвестные типы блоков.

## Как добавить карточку в `cardGrid`
1. Найдите блок типа `cardGrid`.
2. Добавьте элемент в `items`.
3. Заполните `title`, `text`, `order`, `isActive`.

Внутри `cardGrid` карточки также сортируются по `order` и фильтруются по `isActive`.

## Как менять сетку блока
Для `cardGrid`/`exampleGrid` используйте `grid`:
- `auto`
- `2`
- `3`
- `4`

Другие значения не поддерживаются схемой.

## Как менять тему блока
Используйте `theme`:
- `dark`
- `light`
- `graphite`

Другие значения не поддерживаются схемой.

## Что нельзя редактировать через CMS
Через Stage 1 нельзя менять:
- CSS-классы;
- произвольные отступы;
- HTML-разметку;
- произвольные цвета.

## Что дальше
После проверки Stage 1 на странице «Навесы» можно поэтапно переносить другие проектные страницы на тот же подход без редизайна и без изменения архитектуры деплоя.
