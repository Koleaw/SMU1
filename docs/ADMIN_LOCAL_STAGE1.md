# Локальная админка СМУ-1 (Stage A после восстановления)

## Как открыть visual editor
1. Запустите фронтенд: `npm run dev -- --host 127.0.0.1`.
2. Запустите admin API: `npm run admin:api`.
3. Откройте `http://127.0.0.1:4321/SMU1/admin/`.
4. Войдите (`admin / admin`).
5. Откройте страницу: `http://127.0.0.1:4321/SMU1/admin/pages/navesy/`.

## Что доступно в Stage A
После аварийного восстановления доступны:
- стабильное открытие visual editor страницы «Навесы»;
- загрузка `/api/admin/me` и `/api/admin/content/product-sections/navesy`;
- редактирование hero текстом (`heroTitle`, `heroDescription`, `image`, `placeholderLabel`);
- редактирование текстов карточек, строк списков и шагов процесса;
- скрытие/показ карточек и шагов;
- перемещение карточек и шагов вверх/вниз;
- сохранение через `PUT /api/admin/content/product-sections/navesy`.

## Что временно НЕ доступно
В Stage A намеренно отключены:
- upload фото;
- crop;
- add/delete section;
- delete card;
- advanced settings (модальные расширенные настройки).

## Поведение загрузки и ошибок
- При старте показывается `Загрузка данных…`.
- Если сессия не активна, показывается `Требуется вход в админку` и кнопка `В админку`.
- Если API не отвечает за 10 секунд, показывается сообщение о необходимости проверить `npm run admin:api`.
- Runtime ошибки и unhandled promise rejection отображаются в редакторе как видимая ошибка (без белого экрана и бесконечной загрузки).

## Сохранение
Кнопка `Сохранить` отправляет весь draft целиком.
После успешного сохранения:
- статус возвращается к `Нет изменений`;
- показывается подтверждение `Сохранено`.
