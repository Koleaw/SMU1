# Локальная админка СМУ-1 (Stage 1)

## Что изменилось
Главный сценарий Stage 1 теперь визуальный:

1. Вход в админку на `http://127.0.0.1:4321/SMU1/admin/`.
2. Стартовый экран «Админка СМУ-1» с карточками разделов.
3. Основная рабочая карточка — **«Редактировать страницу Навесы»**.
4. Переход в визуальный редактор: `http://127.0.0.1:4321/SMU1/admin/pages/navesy/`.

Технический прежний интерфейс не удален: доступен отдельно как **«Технический режим»** (`/SMU1/admin/technical/`).

## Запуск
1. Терминал 1:
   ```bash
   npm run dev -- --host 127.0.0.1
   ```
2. Терминал 2:
   ```bash
   npm run admin:api
   ```
3. Открыть: `http://127.0.0.1:4321/SMU1/admin/`
4. Логин/пароль: `admin / admin`

## Визуальный редактор «Навесы»

### Верхняя панель
- `← В админку`
- `Редактирование: Навесы`
- Статус: `Нет изменений` / `Есть несохраненные изменения`
- `Сохранить`
- `Открыть публичную страницу`
- `Выйти`

### Как редактировать hero
1. В превью нажмите **«Редактировать hero»**.
2. В правой панели измените:
   - Заголовок (`heroTitle`)
   - Описание (`heroDescription`)
   - Изображение (`image`)
   - Подпись заглушки (`placeholderLabel`)

### Как редактировать блок
1. Каждый `pageBlocks` отображается как отдельная секция с админ-рамкой.
2. На блоке доступны кнопки:
   - `Редактировать блок`
   - `Выше` / `Ниже`
   - `Скрыть` / `Показать`
   - селектор темы (Темная / Светлая / Графитовая)
   - селектор сетки/колонок (если применимо)
3. Для детального редактирования нажмите `Редактировать блок`: откроется правая боковая панель.

### Как редактировать карточки
Для `cardGrid`/`exampleGrid` в боковой панели доступны:
- заголовок карточки;
- описание;
- флаг активности;
- `Выше` / `Ниже`;
- `Скрыть` / `Показать`.

### Как добавить карточку
Нажмите **«Добавить карточку»**. Добавляется карточка:
- `title: "Новая карточка"`
- `text: ""`
- `order: следующий`
- `isActive: true`

### Как менять порядок
- Блоки: кнопки `Выше` / `Ниже` в тулбаре блока.
- Карточки/строки/шаги: `Выше` / `Ниже` в боковой панели.

### Как сохранить
1. После изменений нажмите **«Сохранить»**.
2. Отправляется `PUT /api/admin/content/product-sections/navesy`.
3. При успехе: сообщение `Сохранено`, dirty state сбрасывается без перезагрузки страницы.

## Что работает в Stage 1
- Авторизация: `POST /api/admin/login`, `POST /api/admin/logout`, `GET /api/admin/me`.
- Чтение/запись JSON через local file mode.
- Модель данных `pageBlocks`.
- Визуальное редактирование страницы «Навесы».
- Debug-секция «Показать JSON» внизу.

## Ограничения (пока не реализовано)
- Upload изображений.
- Drag-and-drop.
- Создание/удаление entries.
- Массовый перевод других страниц на визуальный режим.
- Production hosting backend и GitHub write mode.
