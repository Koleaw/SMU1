# Локальная админка СМУ-1 (Stage 1)

## Основной сценарий (visual edit mode)
1. Откройте `http://127.0.0.1:4321/admin/`.
2. Войдите (`admin / admin`).
3. Нажмите карточку **«Редактировать страницу Навесы»**.
4. Откроется visual edit mode: `http://127.0.0.1:4321/admin/pages/navesy/`.

Это основной рабочий путь Stage 1.

## Технический режим
Старый технический редактор сохранен как запасной:
- `http://127.0.0.1:4321/admin/technical/`

По умолчанию он не используется.

## Запуск
1. `npm run dev -- --host 127.0.0.1`
2. `npm run admin:api`
3. Открыть `http://127.0.0.1:4321/admin/`

## Верхняя панель редактора
В visual edit mode сверху фиксированная панель:
- `← В админку`
- `Редактирование: Навесы`
- статус изменений
- `Сохранить`
- `Открыть публичную страницу`
- `Выйти`

## Inline-редактирование hero
В hero есть кнопка **«Редактировать hero»**.

После нажатия редактирование происходит на месте:
- заголовок;
- описание;
- подпись placeholder;
- `Заменить фото` (upload);
- `Убрать фото`;
- `Настроить кадр`;
- `Сбросить` / `Готово` / `Отмена`.

## Настройка кадра (imageView/mediaView)
Для hero и блоков с media доступна кнопка **«Настроить кадр»**.

Открывается inline-панель рядом с выбранной media-зоной (hero или блок):
- редактирование идет прямо в том же прямоугольнике media-зоны (как на публичной странице);
- drag-to-position внутри этого же прямоугольника с ограничением диапазона по текущему масштабу;
- live-preview: кадр меняется сразу во время drag/scale, без ожидания `Сохранить`;
- `fit: cover/contain`;
- `scale` (1..3);
- `positionX` (0..100);
- `positionY` (0..100);
- `Сбросить` / `Готово` / `Отмена`.

Важно:
- это **настройка отображения** (object-fit/object-position/scale);
- физической обрезки файла не происходит;
- upload остается прежним.

## Inline-редактирование карточек
В блоках `cardGrid/exampleGrid` карточки редактируются прямо внутри карточки:
- кнопки `Редактировать`, `Выше`, `Ниже`, `Скрыть/Показать`, `Удалить`;
- в режиме редактирования: заголовок, описание, активность, `Готово`, `Отмена`.

## Добавление и удаление карточек
- Внизу блока есть кнопка **`+ Добавить карточку`**.
- Удаление карточки подтверждается через `confirm`.
- До сохранения изменения находятся только в draft-состоянии.

## Списки и factorList
- `listPanel/factorList`: кнопка `Редактировать список`.
- В edit mode каждая строка показывается отдельной компактной карточкой с кнопками `Ред.`, `↑`, `↓`, `✕`.
- Поддерживаются строки и объекты (неизвестные поля у object item сохраняются).
- Кнопка `+ Добавить строку` внизу списка.

## Process (шаги)
- inline-редактирование шагов;
- `Выше/Ниже/Скрыть`;
- `+ Добавить шаг`.

## Настройки блока (inline рядом с секцией)
В шапке каждого блока кнопка **«Настройки»**.
Открывается inline-панель сразу под шапкой этой же секции:
- Заголовок;
- Intro;
- Тема;
- Сетка (если применимо);
- Колонки (если применимо);
- Активность;
- `Готово` / `Отмена`.

Панель закрывается по `Готово`/`Отмена` и не требует отдельного скролла к низу страницы.

## Добавление/удаление секций
Доступно в visual edit mode:
- нижний блок **«Добавить секцию»** с выбором типа;
- поддерживаются типы: `cardGrid`, `listPanel`, `process`, `factorList`, `mediaText`, `notice`, `exampleGrid`;
- кнопка `Удалить секцию` в шапке каждой секции;
- порядок `order` пересчитывается автоматически;
- JSON на диске меняется только после `Сохранить`.

## Загрузка медиа
Endpoint:
- `POST /api/admin/upload`

Требования:
- только с авторизацией;
- `multipart/form-data`;
- типы фото: `jpg`, `jpeg`, `png`, `webp`, `svg`;
- типы видео: `mp4`, `webm`;
- максимум для фото 10 MB;
- максимум для видео 90 MB;
- файлы сохраняются в `public/uploads`;
- ответ: `{ "path": "/uploads/file-name.ext", "type": "image|video" }`.

Где работает upload:
- hero image;
- hero video на главной странице;
- `mediaText` блоки (поле `media`);
- `process` блоки (поле `media`).

Для главной страницы можно указать два видео:
- **Видео для компьютеров** — обычный крупный файл.
- **Видео для телефонов** — облегченный MP4, который отдается на экранах до 900px.

Для hero video на главной странице рядом с кнопками загрузки есть поле **«Затемнение видео, %»**. Значение `0` убирает темный overlay, `100` оставляет полную силу текущего затемнения.

## Сохранение
Кнопка **«Сохранить»** отправляет:
- `PUT /api/admin/content/product-sections/navesy`

После успеха:
- сообщение `Сохранено`;
- dirty state сбрасывается;
- без полной перезагрузки страницы.

## Публикация
После успешного сохранения visual editor вызывает:
- `POST /api/admin/publish`

Локальный admin API выполняет:
- `git add -- src/content public/uploads`;
- `git commit` только для `src/content` и `public/uploads`;
- `git push`.

После `git push` подключенный к GitHub хостинг запускает автодеплой и публикует новую версию сайта. Если Git не настроен, нет доступа к GitHub или ветка не пушится, изменения остаются сохраненными локально, а админка покажет ошибку публикации.

В верхней панели visual editor есть запасная кнопка **«Опубликовать весь сайт»**. Она вызывает:
- `POST /api/admin/publish-all`

Этот режим делает `git add`, `git commit` и `git push` для основных файлов сайта: `src`, `public`, `docs`, `tools`, конфигов и package-файлов. Он не отправляет `node_modules`, `dist`, `.astro`, `.env*` и log-файлы. Если в редакторе есть несохраненные правки, сначала нужно нажать **«Сохранить»**.

## Debug JSON
JSON доступен только как отладочная секция внизу под `Показать JSON`.

## Ограничения Stage 1

- массовый visual editor для всех страниц;
- production hosting backend.
